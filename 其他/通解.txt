
#==================== 项目结构解析 ====================
# 顶层目录说明：
# 机制/         —— 游戏核心机制与AI逻辑，包含指挥系统、检测、开局、测试等子模块
# 作战单位/     —— 陆军、海军等单位配置，分为地面、海军、远程等子目录
# 空中单位/     —— 飞行单位及其子类型（轰炸机、拦截机、运输机等）
# 建筑/         —— 各类建筑物、资源点、工厂等配置
# 其他/         —— 工具、说明文档、贴图、地图等杂项

#==================== 机制/指挥系统模块分层 ====================
# 指挥系统/           —— 各类中控与区块的AI主逻辑
#   |-- 作战通用模块.ini      # 各类中控的基础行为与战力统计，建议仅做底层通用修改
#   |-- 区块通用模块.ini      # 区块类（如作战区块、海军区块）的通用行为
#   |-- 作战区块.ini          # 单个作战区块的行为、回防、协防逻辑
#   |-- 海军区块.ini          # 海军专用区块，负责生成海军中控
#   |-- 海军作战中控.ini      # 海军主控，聚集、进攻、目标选择等
#   |-- 陆军作战中控.ini      # 陆军主控，聚集、进攻、目标选择等
#   |-- 附属检测系统/         # 各类检测与辅助单元（如集结检测、距离检测、协防指挥等）

#==================== 标签与资源通解 ====================
# 【单位类型标签】
#   地      —— 地面单位（坦克、机甲等），可被对地武器攻击
#   海      —— 海面单位（舰船等），可被对地武器攻击
#   空      —— 飞行单位（拦截机、武装直升机等），可被对空武器攻击
#   水      —— 水下单位（潜艇等），需对水武器攻击
#
# 【区块/中控标签】
#   作战区块      —— 区块AI，负责局部战力检测与回防
#   海军区块      —— 海军专用区块，生成海军中控
#   作战中控      —— 各类中控AI的通用标识
#   陆军作战中控  —— 陆军主控，调度地面部队
#   海军作战中控  —— 海军主控，调度海上部队
#
# 【行为状态标签】
#   聚集        —— 单位处于集结/待命状态，等待进攻或下一步指令
#   开始进攻    —— 单位已满足进攻条件，准备发起进攻
#   发起进攻    —— 单位正在执行进攻行为
#   未下达指令  —— 单位当前无明确指令，通常等待中控分配
#   发起撤退    —— 单位进入撤退流程
#   发起协防    —— 单位响应协防请求，前往支援
#
# 【回防/协防标签】
#   需要回防    —— 区块/建筑检测到威胁，触发协防指挥
#   协防指挥    —— 协防单位已被调度，正在执行支援任务
#
# 【建筑与攻击类型标签】
#   建筑物      —— 所有建筑类型的通用标识
#攻击类型标签：
#   对地，对地170/190/210/240/290/310/390
#    对空，防空（T1级别），中防空（T2级别），大防空（T3级别）
#    拦截，重拦  —— 自身为空中单位，具备对空作战能力
#    对水     —— 具备对水单位的攻击能力
#
# 【特殊/辅助标签】
#   前线        —— 标记前线位置，常用于AI目标选择
#   资源点      —— 资源采集点
#   家矿、中矿、外矿 —— 资源点位置细分
#   结束        —— 行为或流程终止标记
#   计算        —— 临时计算/统计用标签
#
# 【常用资源说明】
#   总战力      —— 当前单位/区块/中控的总战斗力
#   最高战力    —— 历史最大战力，用于进攻/撤退判定
#   军队战力    —— 当前己方部队的战力总和
#   压力值      —— 战场压力评估，用于AI决策
#   冷却        —— 行为冷却计时
#   编号        —— 单位唯一编号，便于遍历与统计
#   陆军/海军   —— 各类部队数量统计
#   对空战力/对地战力 —— 单位/区块对空/对地能力

#==================== 关键交互与注意事项 ====================
# 1. 各中控与区块通过 waypoint（如 follow、attack、move）和标签（如聚集、进攻）进行状态流转
# 2. 检测系统通过 memory 变量（如 memory.附属检测.resource.陆军）向主控反馈统计结果
# 3. 大量行为依赖 requireConditional 条件，注意魔法数与时间片（如 self.resource.时间%15）带来的同步问题
# 4. 修改通用模块（如作战通用模块.ini、区块通用模块.ini）需谨慎，建议先在测试分支验证
# 5. 标签与 waypoint 的添加/清理需成对出现，防止状态卡死或标签泄露
# 6. 推荐在 action_信息 处增加 debug 输出，便于实战排查

#==================== 文件举例说明 ====================
# 机制/指挥系统/陆军作战中控.ini
#   —— 管理陆军聚集、进攻、目标选择，强依赖附属检测与标签流转
# 机制/指挥系统/附属检测系统/集结检测.ini
#   —— 统计中控周边陆军数量，输出 resource.陆军 供主控判定聚集/分散

#   —— 单位属性、武器、标签、AI行为等配置
# 空中单位/拦截/拦截机/轻拦截.ini
#   —— 飞行单位属性、拦截行为、标签

#==================== 维护建议 ====================

# 2. 新增模块建议先写文件头说明，描述用途、关键点、注意事项
# 3. 调整全局行为（如聚集/进攻判定、检测阈值）建议先在测试分支验证
